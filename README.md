# AI 报告自动化工作流系统

脱敏后的公开演示版，展示如何把原本依赖人工串联的报告制作流程，重构为端到端 AI 工作流。

## 项目概述

传统报告生产通常需要在素材整理、章节拆分、文案撰写、图片编排、页面预览和最终导出之间来回切换，流程长、重复操作多，而且很难沉淀为可复用的系统能力。

这个项目的目标不是单纯“用 AI 写一段文案”，而是把整条报告生产链路产品化：用户基于结构化输入和参考素材，在同一套界面中完成内容生成、页面编辑、实时预览、草稿保存和导出交付。

## 项目价值

- 把分散的手工步骤收敛成统一工作流，减少跨工具切换
- 把报告内容拆成可配置章节，支持不同报告类型复用同一套编辑框架
- 把 AI 能力嵌入页面级编辑流程，而不是停留在单次文本生成
- 把预览、草稿和导出整合进同一套交互链路

## 核心能力

- 章节化编辑：按页面和章节组织内容，适合长篇多页报告
- 模板驱动渲染：不同报告类型映射不同模板与字段结构
- AI 接入位：保留文案生成和工作流编排的接入结构
- 实时预览：编辑结果可直接映射到报告页面
- 草稿保存：支持本地保存与恢复，适合长流程编辑
- 导出能力：面向 HTML / PDF 报告交付场景

## 技术实现

- 原生 JavaScript（ES Modules）
- HTML 模板系统
- CSS 页面样式与多版式布局
- LocalStorage / SessionStorage 状态持久化
- 配置驱动的报告结构映射
- 可接入外部 AI 服务与工作流系统

## 项目结构

```text
src/
  app.js                  # 主交互逻辑
  api/                    # 请求层
  config/                 # 报告结构、映射与运行时配置
  services/               # 模板、草稿、上传等服务
  styles/                 # 样式

templates/
  主题-风格企划模板.html
  主题-主题企划模板.html

config.js                 # 报告类型与章节结构配置
index.html                # 应用入口
```

## 本地查看

这个仓库以公开演示为主，可以直接作为静态前端项目查看。

- 使用任意静态服务器打开项目目录即可
- 如需接入你自己的 AI 服务，可补充 `src/config/env.js`
- `prompts/` 与 `workflow/` 在公开版中仅保留说明文件，不包含生产内容
