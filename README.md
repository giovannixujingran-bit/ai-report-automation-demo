# AI 报告自动化 Demo

这是一个把公司内部原本依赖人工完成的报告制作流程，改造成端到端 AI 工作流的前端演示项目。

项目重点不在某个具体行业术语，而在于把原先分散在资料整理、章节组织、文案生成、页面排版、预览导出中的手动步骤，收敛成统一的结构化编辑流程。

## 项目解决了什么问题

在传统报告生产里，常见流程往往是：

- 人工收集和整理参考素材
- 人工拆分章节和页面结构
- 人工撰写标题、关键词、摘要和说明文案
- 人工把图片塞进固定版式
- 最后再手动导出报告

这个项目把这些步骤整合为一个可视化系统：

- 用章节配置驱动页面结构
- 用模板系统承接多页报告展示
- 用 AI 服务承接文案生成与流程编排
- 用本地草稿保存承接中断恢复
- 用预览和导出承接最终交付

## 当前公开版保留了什么

- 多页报告编辑界面
- 报告模板与章节配置
- 本地草稿保存逻辑
- 页面预览与导出相关前端结构
- 公开可展示的前端代码组织方式

## 当前公开版移除了什么

出于安全和知识产权考虑，以下内容没有放入公开版：

- 私有提示词与业务方法论素材
- 生产环境密钥、内网地址与代理配置
- 公司内部使用的完整工作流编排细节
- 示例业务数据与内部产出物

因此，这个仓库更适合作为：

- 项目案例展示
- 前端结构与系统设计说明
- 报告生成类产品的实现参考

而不是一个可直接连接生产环境运行的完整版本。

## 核心能力

- 章节化编辑：按页面组织内容，而不是按长文本堆叠
- 模板驱动渲染：不同报告类型映射不同页面模板
- AI 流程接入位：保留 AI 生成的调用结构与前端编排位置
- 实时预览：编辑内容可以同步进入报告预览
- 草稿保存：支持本地自动保存与恢复
- 导出能力：面向 HTML / PDF 报告交付场景

## 项目结构

```text
src/
  app.js                  # 主交互逻辑
  api/                    # 请求层
  config/                 # 报告结构、映射与运行时配置
  services/               # 模板、草稿、上传等服务
  styles/                 # 样式

templates/
  主题-风格企划模板.html
  主题-主题企划模板.html

config.js                 # 报告类型与章节结构配置
index.html                # 应用入口
```

## 适合展示的点

- 如何把复杂业务流程抽象成结构化前端系统
- 如何用配置驱动多种报告模板
- 如何把 AI 生成接入编辑器工作流，而不是孤立做一个聊天框
- 如何处理长篇可视化内容的预览、草稿和导出

## 本地查看

公开版默认以前端结构展示为主。

- 可以直接作为静态站点启动查看界面
- 若要接入你自己的 AI 服务，请自行补充 `src/config/env.js`
- `prompts/` 与 `workflow/` 中的文件在公开版中仅保留说明，不含生产内容
